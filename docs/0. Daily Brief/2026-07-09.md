### Content of the Day — TTP: Kerberoasting (enumeration → offline crack), MITRE **T1558.003**

Kerberoasting is the cleanest "any-authenticated-user → credential access" move in AD, and it maps directly to the Thursday enum→exploit shape. The mechanic: any domain principal can request a TGS (service ticket) for any account that has a `servicePrincipalName` set. The encrypted portion of that ticket is sealed with a key derived from the *service account's password* — so if you can pull it in RC4 (etype 23), you crack it offline with zero further interaction against the DC. The enumeration half is an LDAP query for `(&(samAccountType=805306368)(servicePrincipalName=*))` — deliberately scoped to **user** objects, because computer accounts also have SPNs but their 120-char machine passwords are effectively uncrackable. You're hunting human-managed service accounts (SQL, IIS, backup agents) that all too often carry `Password123!`-tier secrets and, worse, Domain Admin membership.

Exploitation, from Linux: `impacket-GetUserSPNs -request -dc-ip <DC> <domain>/<user>:<pass>` dumps `$krb5tgs$` hashes straight to your terminal. From a Windows foothold, `Rubeus.exe kerberoast /nowrap /outfile:h.txt`. Then crack offline: `hashcat -m 13100 h.txt rockyou.txt -r best64.rule`. The nuance that keeps this CPTS-level rather than script-kiddie: modern DCs increasingly hand back **AES** tickets (etype 17/18 → hashcat 19600/19700), which are *dramatically* slower to crack, so smart operators use `Rubeus kerberoast /tgtdeleg` to negotiate an RC4-downgraded ticket where the config allows. That same downgrade is also your loudest IOC — a burst of TGS-REQs with encryption type `0x17` lighting up Event ID 4769 is the canonical detection, so roast selectively, not the whole domain.

The genuinely nuanced variant worth internalizing is **targeted Kerberoasting**: if BloodHound shows you a `GenericWrite`/`GenericAll` edge over a user that has *no* SPN, you write a bogus one, roast it, then clean up — `Set-DomainObject -Identity victim -Set @{serviceprincipalname='fake/svc'}` (PowerView), roast, then null it back out. That turns an ACL edge into an offline password-recovery primitive, bridging cleanly into the ACL-abuse tradecraft (Shadow Credentials, GPO abuse) from earlier this week.

OSCP/CRTP tie-in: this fires on essentially every OSCP AD set and CRTP lab — grab any low-priv domain cred, roast, crack a weak service account, and you frequently leapfrog straight to tier-0 because service accounts are chronically over-privileged.
