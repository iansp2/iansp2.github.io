### Content of the Day — Wildcard / Argument Injection (the `*` privesc trick)

A classic, slightly evil Saturday trick: turning shell **glob expansion** into **argument injection**. When a privileged process runs something like `tar cf /backup.tar *`, `chown -R app *`, or `rsync * dest/` inside a directory you can write to, the shell expands `*` into the *filenames* in that directory **before** the binary runs. If you create files whose names begin with `-`, those filenames get handed to the program as **command-line options**, not data. The program never knows the difference — and if it's running as root out of a cron job, so much the better. The defensive fix is boringly simple (`tar cf backup.tar -- *` or `./*`), which is exactly why you still find this everywhere.

The canonical payload abuses GNU `tar`'s checkpoint hooks. In a writable directory that a root cron runs `tar cf ... *` against, drop a script plus two option-shaped filenames:

```bash
echo 'cp /bin/bash /tmp/rb && chmod +s /tmp/rb' > runme.sh
touch -- '--checkpoint=1'
touch -- '--checkpoint-action=exec=sh runme.sh'
```

When the glob expands, `tar` sees `--checkpoint=1 --checkpoint-action=exec=sh runme.sh` as flags and executes `runme.sh` as root — dropping a SUID `bash` you trigger with `/tmp/rb -p`. Note the `touch --`: the `--` terminator stops *touch itself* from parsing those leading dashes so the file actually gets created. Same family, different binaries: `chown`/`chmod` honor `--reference=FILE` (create a file named `--reference=/tmp/owned` to copy arbitrary ownership/permissions onto the target set), and `rsync` will run a command via `-e`/`--rsh=`. GTFOBins documents the exploitable flags for each. This is a privilege-escalation vector, typically reached via a cron job (**MITRE ATT&CK T1053.003 – Scheduled Task/Job: Cron**) and landing as arbitrary command execution (**T1059**).

Where you'll hit it: this is a Proving Grounds and OSCP lab staple — a `pspy`/`crontab` sweep reveals a root task globbing over a directory you own, and the whole box hinges on recognizing that `*` is attacker-controlled input. It's conceptually the same lesson as today's **GhostApproval** and **OpenClaw** items above: a trusted process interpreting attacker-controlled *names* (files, symlinks, mount paths) with more authority than intended.

### One Thing to Try

In an authorized lab with a root cron running `tar cf ... *`: `echo 'cp /bin/bash /tmp/rb; chmod +s /tmp/rb' > x.sh; touch -- '--checkpoint=1'; touch -- '--checkpoint-action=exec=sh x.sh'` — then wait for the job and run `/tmp/rb -p`.
