### Content of the Day — TTP: Coercion → NTLM Relay (Adversary-in-the-Middle, MITRE T1557.001)

The whole point of relaying is that you never touch the hash. NTLM authentication is a challenge/response you can *forward live* to a third party that will accept it — so instead of capturing NetNTLMv2 and grinding it in hashcat, you pass the victim's authentication straight into a target and act as them. The exploitation loop is three moves: **find a relay target, coerce authentication toward yourself, forward it and cash out.** Enumeration is where it starts — you need somewhere that accepts relayed auth. `nxc smb <range> --gen-relay-list targets.txt` gives you every host **without SMB signing required**; DCs that don't **enforce LDAP/LDAPS signing** are relay-to-LDAP candidates; and an **AD CS web-enrollment endpoint** (`/certsrv`) is the ESC8 jackpot. No target that accepts unsigned relayed auth, no attack — signing is the whole ballgame.

Now you need authentication to *arrive*. On the passive end, **Responder** poisons LLMNR/NBT-NS/mDNS on segments where name resolution falls back to broadcast (fat-finger a share name and a box coughs up creds), and **mitm6** hijacks IPv6 DNS via rogue WPAD — brutal on default dual-stack networks nobody hardened for v6 (**T1187 Forced Authentication**). To actively pull *machine-account* auth — which authenticates at SYSTEM tier and is far more valuable — you coerce: **PetitPotam** (MS-EFSRPC), the PrinterBug (MS-RPRN), MS-DFSNM, and friends, all wrapped by **Coercer**, which sprays every coercion RPC at a target until one bites and fires an authentication back at your listener.

The payoff is entirely about *where you relay*. `ntlmrelayx.py` is the workhorse. Relay to **LDAP/LDAPS** and you can write **RBCD** (`--delegate-access`, dropping `msDS-AllowedToActOnBehalfOfOtherIdentity`) or add **Shadow Credentials** (`--shadow-credentials`) — both covered earlier this week; relay is simply the mechanism that *delivers* them. The heavier chain is **ESC8**: relay a coerced **DC machine-account** auth to the AD CS HTTP enrollment endpoint (`-t http://<ca>/certsrv/certfnsh.asp --adcs --template DomainController`), get a certificate *as the DC*, then PKINIT that cert to a TGT (`gettgtpkinit.py` / Rubeus) and **DCSync** the domain (**T1649 Steal or Forge Authentication Certificates → T1003.006 DCSync**). That's coercion-to-domain-admin with no password ever cracked.

**OSCP/lab tie-in:** this is bread-and-butter internal AD. On HTB/PG boxes where SMB signing is off, the intended path is frequently Responder-capture-then-relay; the CPTS AD modules drill the full mitm6 + `ntlmrelayx` + coercion chain. Nuance worth internalizing: relay only works where signing isn't enforced *and* you have a coercion/poisoning primitive — if signing is hard everywhere and v6/WPAD is locked down, you drop back to capturing NetNTLMv2 and cracking offline (`hashcat -m 5600`).
