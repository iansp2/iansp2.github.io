### Content of the Day — CTF-style trick: PHP filter chains → RCE

You've found `include($_GET['page'])` or `file_get_contents($_GET['file'])` with no path filtering — a textbook LFI. Classic OSCP moves are log poisoning, `/proc/self/environ`, or session files, but sometimes there's nothing to poison: logs aren't readable, no upload, no session foothold. This is where **PHP filter chains** turn the LFI into a full webshell using *nothing but the PHP engine itself* — no file ever touches disk. The primitive maps to **MITRE T1190 (Exploit Public-Facing Application)**, with the payoff being **T1059** command execution.

The mechanism: PHP's `php://filter/` wrapper lets you stack conversion filters on a stream before it's read. The `convert.iconv.*` filters transcode between character encodings, and — crucially — certain iconv conversions emit predictable extra bytes. By chaining hundreds of these conversions in the right order, you can *synthesize arbitrary leading bytes* in the decoded output out of thin air. So even if `resource=php://temp` starts empty, the filter chain generates a fully attacker-controlled prefix. Point that at an `include()` sink and you've made PHP `include` a string you built byte-by-byte through encoding math — i.e. `<?php system($_GET[0]); ?>`. (The same base64-decode/dechunk trick, the original loknop/Gynvael technique, also gives you a blind error-oracle file read when you only have `file_get_contents` and no output.)

You don't hand-assemble the chain — Synacktiv's `php_filter_chain_generator.py` does the encoding search for you. Generate the chain, drop the resulting `php://filter/...` blob into the vulnerable param, and append your command. It's elegant because it sidesteps every "there's no file to include" dead end: the payload *is* the path. Where you'll hit it: HTB web boxes and PortSwigger's LFI labs, and any real engagement where you've got an `include`/`require` sink over user input but the environment is too locked down for log poisoning. It's the trick that converts a "meh, LFI, moving on" into a shell.

One caveat for CPTS-tier nuance: the chain only helps when the sink actually *executes* PHP (`include`/`require`/`eval`-adjacent). A pure `file_get_contents` that echoes back gives you arbitrary read, not RCE — still valuable (read `wp-config.php`, `.env`, private keys), but know which primitive you have before you burn time.
